UDP Socket Programming Project – Computer Networks
Project Overview
This project implements a client–server communication system using UDP sockets in PHP, with an additional HTTP monitoring server running in parallel.
Each team member developed separate modules that work together to provide functionality similar to a basic file‑management and monitoring system on a network.

Architecture and Components
1. Server (server.php)
The UDP server handles communication with clients and manages user sessions, permissions, message logging, and timeout handling.

Main features:

Binds to IP 0.0.0.0 and port 5000.
Accepts up to 4 simultaneous clients ($max_clients = 4).
Manages login and user roles (admin and user).
Logs all messages into clients.log, commands.log, and messages.log.
Removes inactive clients after 30 seconds (timeout mechanism).
Works in parallel with the HTTP server (stats.php).
Saves statistics to data/stats.json.
2. Client (client.php)
Each client connects to the UDP server, sends messages, receives responses, and executes commands according to its role.

Client roles and behavior:

Admin (write, read, execute):
Can perform all available commands listed below.

User (read only):
Can only execute /list (list all files) and /read <filename> (read file content).
Any other attempt to use restricted commands returns "Permission denied".

Implemented commands for admin:

Command	Description
/list	Lists all files in the server directory
/read <filename>	Reads the content of the specified file
`/upload	`
/download <filename>	Downloads a file from the server
/delete <filename>	Deletes a file from the server
/search <keyword>	Searches files containing the given keyword
/info <filename>	Displays file size, creation, and modification date
Clients connect by defining:
php


$serverIP = "127.0.0.1";
$serverPort = 5000;
Messages are sent and received in text form (username|message).
Admin clients have a faster response time than regular users (users have an artificial delay using usleep(500000)).
3. HTTP Monitoring Server (stats.php)
A lightweight TCP server running in parallel on port 8080 using the HTTP protocol. It allows the server administrator to monitor server activity through a web browser by visiting:



[localhost](http://localhost:8080/stats)
Features:

Returns server statistics in JSON format including:
Active clients
IP addresses and ports
Total message count
Recent messages log
Server timestamp
Uses data from log files generated by the main UDP server.
Example output:

json


{
  "status": "OK",
  "server_time": "2026-04-19 13:29:55",
  "active_clients": 2,
  "ip_addresses": ["127.0.0.1:50249", "127.0.0.1:50250"],
  "messages_count": 14,
  "messages": [...]
}
4. Commands Module (commands.php)
A helper module that implements all file-related commands used by the server.

Supported operations:

listFiles($path)
readFileContent($path, $filename)
uploadFile($path, $filename, $content)
downloadFile($path, $filename)
deleteFile($path, $filename)
searchFiles($path, $keyword)
fileInfo($path, $filename)
All file paths are sanitized to prevent path traversal vulnerabilities.
